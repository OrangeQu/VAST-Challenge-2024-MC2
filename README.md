# FishEye Analytics - Q1 船舶-货物关联分析

## 📋 项目概述

本项目针对 Oceanus 渔业数据，开发了一套船舶-货物概率关联系统，通过将港口出口记录与船舶停留数据进行交叉比对，识别船舶与货物之间的关联关系，揭示季节性运输模式，并检测潜在的异常行为（如非法捕捞、未报告卸货等）。

---

## 🆚 改进说明

### 1. 桑基图 (Sankey Diagram) - 显著增强

| 维度 | 原始版本 | 改进版本 |
|------|----------|----------|
| 布局算法 | 自定义简化布局，节点位置可能重叠 | 使用标准 `d3-sankey` 库，自动优化布局 |
| 筛选功能 | 无筛选能力 | 支持按鱼种 TopN、船舶 TopN、月份范围、最小吨位筛选 |
| 交互高亮 | 无悬停高亮 | 悬停节点/连线时高亮关联路径，其他元素变暗 |
| 节点显示 | 可能显示孤立节点 | 支持隐藏孤立节点，聚焦核心流向 |
| 标签截断 | 简单截断 | 智能截断（遇到 `/` 优先截断） |

**影响**：用户可以动态探索不同维度下的货物流向，快速识别核心运输路径。

---

### 2. 热力图 (Heatmap) - 全新升级

| 维度 | 原始版本 | 改进版本 |
|------|----------|----------|
| 数据类型 | 港口 × 月份 | 鱼种 × 月份（更精细的货物分析维度） |
| 港口筛选 | 无 | 支持按港口筛选，对比不同港口的鱼种季节模式 |
| TopN 控制 | 无 | 滑块 + 数字输入，控制显示的鱼种数量（3-10种） |
| 色阶类型 | 线性 | 线性 + 对数色阶（适配数据跨度大的场景） |
| 旺季标注 | 无 | 自动高亮 8-11 月旺季区域 |
| 单元格数值 | 仅颜色 | 大单元格内直接显示数值 |
| 洞察更新 | 静态文本 | 根据筛选条件动态生成分析洞察 |

**影响**：用户可以深入分析特定港口的鱼种季节分布，发现 South Paackland 在 3-4 月的异常波动。

---

### 3. 弦图 (Chord Diagram) - 模式切换与高亮

| 维度 | 原始版本 | 改进版本 |
|------|----------|----------|
| 显示模式 | 固定 Top5 | 支持 Top5 / 全部 两种模式切换 |
| 交互高亮 | 无 | 悬停弦或节点时高亮关联关系，其他元素变暗 |
| 标签优化 | 可能重叠 | 动态调整标签位置，支持旋转 |
| 洞察内容 | 静态 | 根据模式动态更新分析文本 |
| 颜色方案 | 固定配色 | 港口与鱼种使用不同色系，便于区分 |

**影响**：用户可以切换视图全面了解货物流向结构，并通过悬停交互快速定位特定港口-鱼种组合。

---

### 4. 新增图表：船舶停留 vs 货物匹配异常检测

| 维度 | 原始版本 | 改进版本 |
|------|----------|----------|
| 存在性 | ❌ 不存在 | ✅ **全新功能** |
| 核心指标 | - | 红色柱 = 船舶停留次数，蓝色折线 = 有货物记录支持的停靠 |
| 港口选择 | - | 支持按港口筛选 |
| 异常识别 | - | 自动标记差异最大的月份 |
| 数据来源 | - | 整合 AIS 停留数据与交付报告的匹配结果 |

**影响**：**这是本次改进的核心价值** — 直接识别“船在港口停靠但没有匹配货物记录”的异常情况。例如，如果某港口在特定月份有大量船舶停留但缺少对应的货物交付记录，可能暗示：
- 未报告的卸货活动
- 数据采集缺口
- 潜在的 IUU（非法、不报告、不管制）捕捞行为

### 5. 独立时间线视图 (timeline.html) - 全新页面

| 维度 | 原始版本 | 改进版本 |

- 识别旺季前的船舶集结模式

## 🚀 使用说明

### 环境要求
- 现代浏览器（Chrome / Firefox / Edge 最新版本）
- 本地 HTTP 服务器（由于 CORS 限制，不能直接双击 HTML 文件打开）

### 启动方式

#### 方法一：使用 Python 简易服务器
```bash
# 进入项目根目录
cd /path/to/FishEye-Analytics

# Python 3
python -m http.server 8080

# 或 Python 2
python -m SimpleHTTPServer 8080
```
然后在浏览器访问 `http://localhost:8080`

#### 方法二：使用 VS Code Live Server
- 安装 Live Server 插件
- 右键点击 `index.html` → `Open with Live Server`

### 页面导航

| 页面 | 路径 | 说明 |
|------|------|------|
| 首页 | `index.html` | 项目总览入口 |
| Q1 分析 | `q1.html` | 船舶-货物关联主分析页 |
| 时间线视图 | `timeline.html` | 独立时间线分析页 |

### Q1 页面使用指南

#### 桑基图操作
1. **鱼种筛选**：选择 TopN 数量，仅显示出货量最高的 N 种鱼
2. **船舶筛选**：选择 TopN 数量，仅显示关联吨位最高的 N 艘船
3. **月份范围**：选择起始/结束月份，限定分析时间窗口
4. **悬停交互**：鼠标悬停任意节点或连线，高亮相关路径
5. **孤立节点**：勾选“显示孤立节点”可查看无连线的实体

#### 热力图操作
1. **港口筛选**：选择特定港口，查看该港口的鱼种×月份分布
2. **鱼种数量**：拖动滑块控制显示的 TopN 鱼种数
3. **色阶类型**：
   - 线性色阶：适合数据分布均匀的场景
   - 对数色阶：适合数据跨度大的场景（如少数月份异常高）
4. **洞察区域**：自动更新分析结论，提示异常波动

#### 弦图操作
1. **模式切换**：
   - Top5 模式：显示 Top5 港口 × Top5 鱼种，聚焦核心关系
   - 全部模式：显示所有港口和鱼种，全面了解结构
2. **悬停高亮**：悬停任意弦或节点，高亮所有关联路径

#### 异常检测图操作
1. **港口选择**：切换不同港口，观察船货匹配差异
2. **数据解读**：
   - 红色柱高度 = 船舶停留次数
   - 蓝色折线 = 有货物记录支持的停靠次数
   - 两者差值越大 → 异常程度越高
3. **典型异常场景**：
   - South Paackland 3-4 月：停留多但匹配少 → 未报告卸货嫌疑
   - 保护区内停留但无货物记录 → IUU 捕捞嫌疑

### 时间线视图使用指南

1. **筛选联动**：
   - 船舶筛选：限定特定船舶
   - 鱼种筛选：限定货物鱼种
   - 地点筛选：支持常规港口 + 保护区/渔区
   - 月份/范围：快速定位异常期（3-4月）或旺季（8-10月）

2. **三层视图解读**：
   - **上层（船舶停留）**：
     - 选择港口时：显示多船停留时间条
     - 选择“全部地点”时：显示港口停留热力图
   - **中层（港口报告）**：
     - 气泡大小 = 吨位，颜色 = 鱼种
     - 气泡自动防重叠布局
   - **下层（交易趋势）**：
     - 紫色面积 + 折线 = 每日交易总量
     - 紫色虚线 = 日均线

3. **区域颜色编码**：
   - 🔴 红色：保护区（Ghoti Preserve）- 非法进入嫌疑
   - 🟠 橙色：渔区（Wrasse Beds）- 可疑频繁停留
   - 🔵 蓝色：常规港口

## 🔍 关键业务洞察

### 正常季节性模式
- **旺季**：8-11 月，出货量占全年的主要部分
- **淡季**：2-7 月，出货量相对低迷

### 核心枢纽
- **South Paackland**：最大出口港口，主导 Cod 出口
- **Paackland**：Tuna 出口专业港
- **Lomark**：多样化出口特征

## 🛠️ 技术栈

- **D3.js v7**：核心可视化库
- **d3-sankey**：桑基图布局算法
- **原生 JavaScript**：无额外框架依赖
- **响应式设计**：支持窗口 resize 自适应
